# WAS 이슈 재현 테스트 애플리케이션

Web/WAS 환경에서 자주 발생하는 이슈들을 의도적으로 재현할 수 있는 테스트 애플리케이션입니다.

## ⚠️ 경고

**이 애플리케이션은 운영 환경에서 절대 사용하지 마세요!**

- 서버 다운, 성능 저하, 데이터 손실 등의 위험이 있습니다.
- **반드시 테스트/개발 환경에서만 사용**하시기 바랍니다.
- 각 테스트는 실제 운영 이슈를 재현하므로 서버에 심각한 영향을 줄 수 있습니다.

## 📋 요구사항

- **JDK**: Java 8 이상
- **WAS**: WebLogic, Tomcat, JBoss, JEUS 등 JSP를 지원하는 모든 WAS
- **권장**: 테스트 전용 서버 환경

## 📁 프로젝트 구조

```
.
├── WEB-INF/
│   └── web.xml          # 웹 애플리케이션 설정 파일
├── index.jsp            # 메인 페이지 (테스트 목록)
├── oome.jsp             # OutOfMemoryError 재현
├── connection_pool.jsp  # 커넥션 풀 고갈 재현
├── slow_query.jsp       # 느린 쿼리/타임아웃 재현
├── session_leak.jsp     # 세션 누수 재현
├── thread_pool.jsp      # 스레드 풀 고갈 재현
├── file_handle.jsp      # 파일 핸들러 누수 재현
├── cache_leak.jsp       # 캐시 누수 재현
├── permgen_leak.jsp     # PermGen/Metaspace 누수 재현
├── deadlock.jsp         # 데드락 시뮬레이션
├── cpu_overload.jsp     # CPU 과부하 재현
├── error.jsp            # 에러 페이지
└── README.md            # 이 문서
```

## 🚀 배포 방법

### 1. WAR 파일로 배포

```bash
# 프로젝트 디렉토리를 WAR로 압축
jar cvf was-issue-test.war *

# 또는 빌드 도구 사용
mvn clean package
```

### 2. 디렉토리로 배포

프로젝트 전체를 WAS의 웹 애플리케이션 디렉토리에 복사:

- **Tomcat**: `$CATALINA_HOME/webapps/was-issue-test/`
- **WebLogic**: 도메인 설정에서 디렉토리 경로 지정
- **JBoss**: `$JBOSS_HOME/standalone/deployments/was-issue-test.war` 또는 디렉토리

### 3. 접속

배포 후 브라우저에서 접속:
```
http://localhost:8080/was-issue-test/
```

## 🧪 테스트 항목

### 1. OutOfMemoryError (OOME) 재현
**파일**: `oome.jsp`

- **목적**: 힙 메모리를 고의로 소진시켜 OOME 발생
- **재현 방법**: 대용량 바이트 배열을 반복적으로 할당
- **관찰 포인트**: 
  - 힙 메모리 사용량 증가
  - GC 빈번 발생
  - 최종적으로 OOME 발생

### 2. 커넥션 풀 고갈
**파일**: `connection_pool.jsp`

- **목적**: 데이터베이스 커넥션을 최대치까지 사용하여 풀 고갈 재현
- **재현 방법**: 
  - JDBC 커넥션을 반복적으로 생성
  - 커넥션을 닫지 않고 유지
- **관찰 포인트**:
  - "Maximum connections reached" 에러
  - "Connection pool exhausted" 에러
  - 새로운 요청 처리 불가

### 3. 느린 쿼리/타임아웃
**파일**: `slow_query.jsp`

- **목적**: 장시간 실행되는 쿼리를 수행하여 타임아웃 재현
- **재현 방법**:
  - DB별 SLEEP 함수 사용 (Oracle: DBMS_LOCK.SLEEP, MySQL: SLEEP, PostgreSQL: pg_sleep)
  - 또는 직접 쿼리 입력
- **관찰 포인트**:
  - Query timeout 발생
  - Connection timeout 발생
  - 트랜잭션 타임아웃

### 4. 세션 누수
**파일**: `session_leak.jsp`

- **목적**: 세션에 대용량 데이터를 저장하여 세션 누수 재현
- **재현 방법**: 세션에 반복적으로 대용량 바이트 배열 저장
- **관찰 포인트**:
  - 세션 메모리 사용량 증가
  - 세션 저장소 부족
  - 서버 메모리 부족

### 5. 스레드 풀 고갈
**파일**: `thread_pool.jsp`

- **목적**: 대량의 스레드를 생성하여 스레드 풀 고갈 재현
- **재현 방법**: 대량의 데몬 스레드 생성 및 유지
- **관찰 포인트**:
  - "Thread pool exhausted" 에러
  - "Maximum thread count reached" 에러
  - 새로운 요청 처리 불가

### 6. 파일 핸들러 누수
**파일**: `file_handle.jsp`

- **목적**: 파일 핸들을 열고 닫지 않아 파일 핸들러 누수 재현
- **재현 방법**: FileInputStream을 반복적으로 생성하되 close() 호출 안 함
- **관찰 포인트**:
  - "Too many open files" 에러
  - 파일 핸들러 한계 도달
  - 새로운 파일 열기 실패

### 7. 캐시 누수
**파일**: `cache_leak.jsp`

- **목적**: 정적 캐시에 무한정 데이터를 추가하여 메모리 누수 재현
- **재현 방법**: 애플리케이션 컨텍스트에 대용량 데이터를 계속 추가
- **관찰 포인트**:
  - 힙 메모리 지속적 증가
  - GC로도 회수되지 않음
  - 서버 재시작 전까지 유지

### 8. PermGen/Metaspace 누수
**파일**: `permgen_leak.jsp`

- **목적**: 동적 클래스 로딩을 통한 PermGen/Metaspace 누수 재현
- **재현 방법**: 클래스 메타데이터를 반복적으로 생성
- **관찰 포인트**:
  - Java 7 이하: "PermGen space" OOME
  - Java 8+: "Metaspace" OOME
  - 클래스 로딩 실패

### 9. 데드락 시뮬레이션
**파일**: `deadlock.jsp`

- **목적**: 여러 스레드가 서로의 리소스를 기다리는 데드락 재현
- **재현 방법**: 
  - 스레드 1: lock1 획득 → lock2 대기
  - 스레드 2: lock2 획득 → lock1 대기
- **관찰 포인트**:
  - 스레드 영구 블로킹
  - 응답 지연
  - 스레드 덤프에서 데드락 확인 가능

### 10. CPU 과부하
**파일**: `cpu_overload.jsp`

- **목적**: 무한 루프를 통한 CPU 100% 사용 재현
- **재현 방법**: CPU 집약적인 계산을 수행하는 스레드 다수 생성
- **관찰 포인트**:
  - CPU 사용률 100%
  - 서버 응답 지연
  - 다른 요청 처리 불가

## 📊 모니터링 방법

각 테스트 실행 시 다음을 모니터링하세요:

### 1. JVM 메모리
```bash
# jstat 사용
jstat -gc <pid> 1000

# 또는 JVM 옵션으로 힙 덤프
-XX:+HeapDumpOnOutOfMemoryError
-XX:HeapDumpPath=/path/to/dump
```

### 2. 스레드 상태
```bash
# 스레드 덤프
jstack <pid>

# 또는 kill -3 <pid>
```

### 3. 파일 핸들러
```bash
# Linux
lsof -p <pid> | wc -l

# 프로세스별 열린 파일 수 확인
```

### 4. 커넥션 풀
- WAS 관리 콘솔에서 커넥션 풀 모니터링
- 또는 JMX를 통한 모니터링

## 🐛 문제 해결

### JSP 컴파일 에러
- JDK 버전 확인 (Java 8 이상 필요)
- WAS의 JSP 컴파일러 설정 확인

### 권한 에러
- 파일 핸들러 테스트: 임시 디렉토리 쓰기 권한 확인
- 커넥션 풀 테스트: DB 연결 권한 확인

### 메모리 부족으로 테스트 실행 불가
- JVM 힙 크기 증가: `-Xmx2g` 등
- 또는 더 작은 값으로 테스트
